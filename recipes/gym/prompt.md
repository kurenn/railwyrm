Use this prompt in Codex/Claude against a newly generated Railwyrm app to test the Gym recipe behavior.

```text
You are implementing a Gym Management overlay in an existing Rails app generated by Railwyrm.

Context:
- Rails app includes Tailwind, PostgreSQL, RSpec, Devise, Active Storage, ActionText, untitled_ui, and claude-on-rails.
- Use recipes/gym/recipe.yml as source of truth.

Goals:
1) Implement core models for members, visits, plans, class sessions, and bookings.
2) Add Pundit authorization baseline for gym workflows.
3) Deliver responsive pages for dashboard, members, visits, class sessions, and reports.
4) Add seed data for realistic local demo.
5) Keep implementation deterministic and maintainable.

Execution:
- Summarize recipe intent
- Implement models/migrations/enums/associations
- Implement controllers/routes/policies
- Implement views and reusable components
- Run `bundle exec rspec`, `bin/rails routes`, and `bin/rails zeitwerk:check`
- Return concise changelog and remaining TODOs
```
