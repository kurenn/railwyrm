## Test Setup

1. Install dependencies: `bundle install`
2. Create and migrate DB: `bin/rails db:prepare`
3. Seed demo data: `bin/rails db:seed`
4. Start app: `bin/dev`
5. Sign in at `/users/sign_in` with:
   - Email: `admin@acme-recruiting.test`
   - Password: `Password123!`

## Role-Based Scenarios

### Admin scenario

1. Sign in as `admin@acme-recruiting.test`.
2. Visit `/` and confirm dashboard loads.
3. Navigate to `/reports` and `/job_postings`.
4. Confirm no authorization errors are raised.

### Recruiter scenario

1. Sign in as `recruiter@acme-recruiting.test`.
2. Open `/candidates`.
3. Open one candidate profile and verify timeline + actions render.
4. Open `/pipeline` and verify stages are visible.

### Hiring manager scenario

1. Sign in as `hiring.manager@acme-recruiting.test`.
2. Open `/job_postings` and a specific posting.
3. Open `/reports` and verify metrics render.

### Interviewer scenario

1. Create interviewer user in console:
   `bin/rails runner 'User.find_or_create_by!(email: "interviewer@acme-recruiting.test") { |u| u.password = "Password123!"; u.password_confirmation = "Password123!" }'`
2. Sign in as `interviewer@acme-recruiting.test`.
3. Open `/pipeline` and `/candidates`.
4. Confirm pages render and session/auth flow works.

## Functional Smoke Scenarios

1. `bin/rails routes | rg "ats|careers"` should list ATS and careers routes.
2. `bin/rails zeitwerk:check` should pass.
3. `bundle exec rspec` should pass.
4. `bin/rails runner 'puts [Company.count, JobPosting.count, Candidate.count, Application.count].join(",")'` should print non-zero counts after seeding.
5. Visit `/careers` while signed out and confirm public access.
6. Visit `/job_postings` while signed out and confirm redirect to sign in.
