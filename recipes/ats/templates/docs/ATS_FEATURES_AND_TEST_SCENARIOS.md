# ATS Features and Test Scenarios

This app was generated by Railwyrm using the ATS reference recipe.

## Test Setup

1. Install dependencies: `bundle install`
2. Create and migrate DB: `bin/rails db:prepare`
3. Seed demo data: `bin/rails db:seed`
4. Start app: `bin/dev`
5. Sign in at `/users/sign_in` with:
   - Email: `admin@acme-recruiting.test`
   - Password: `Password123!`

## Manual Scenarios

### Scenario A: Recruiter creates and publishes a job

1. Sign in as `recruiter@acme-recruiting.test`.
2. Go to `/job_postings/new`.
3. Fill title, department, location, employment type, description, requirements.
4. Click `Save and publish`.
5. Confirm job appears at `/job_postings` with `Open` status.

### Scenario B: Recruiter creates candidate and application

1. Go to `/candidates/new` and create a candidate.
2. Open the candidate profile.
3. In `Create application`, choose a job and submit.
4. Confirm redirect to application detail and `Application created` message.

### Scenario C: Hiring team moves application through pipeline

1. Open `/pipeline`.
2. Pick the job from the selector.
3. Move a candidate to a new stage.
4. Open the application detail and change status.
5. Confirm updates are persisted after refresh.

### Scenario D: Interview and feedback flow

1. Open an application detail page.
2. Schedule an interview.
3. Submit feedback with a score.
4. Confirm feedback appears in the feedback list.

### Scenario E: Offer and hiring flow

1. On an application page, create an offer.
2. Update offer status to `accepted`.
3. Confirm application transitions to `Hired`.

### Scenario F: Public careers application

1. Sign out.
2. Open `/careers` and then a job detail page.
3. Submit the public application form.
4. Confirm redirect with `Application submitted` notice.
